# Luma Recorder Community Plan

## Community Home

Primary:

- GitHub Issues
- GitHub Discussions

Optional later:

- Discord for live community support
- Telegram for Arabic-speaking users and fast feedback

Start with GitHub only. Add Discord or Telegram after there is repeated community demand.

## Early Community Strategy

### Respond Fast

- Reply to all issues within 24 hours during launch week.
- Label issues clearly:
  - `bug`
  - `feature-request`
  - `teaching-workflow`
  - `translation`
  - `good first issue`
  - `ui-improvement`

### Feature Early Users

Create a monthly section in Discussions:

```text
Educator spotlight: how are you using Luma Recorder?
```

Feature:

- teachers recording mini-lessons
- trainers creating onboarding clips
- course creators recording modules
- support teams recording walkthroughs

### Accept Contributions Quickly

Early contribution speed matters more than perfect process.

- Merge small documentation improvements quickly.
- Accept translation fixes quickly.
- Convert good feature requests into tracked issues.
- Thank contributors by name in weekly updates.

## Contribution Hooks

### Translation Contributions

Ask for:

- Arabic wording improvements
- new languages
- teaching-specific vocabulary
- RTL interface feedback

Issue title:

```text
Help improve Luma Recorder translations
```

### Feature Requests

Prompt users with:

```text
What do you record most often?

- lessons
- course modules
- bug reports
- onboarding videos
- software tutorials
- student feedback
```

### UI Improvements

Invite:

- better icons
- clearer Arabic labels
- simpler teaching workflow
- better recording status
- improved first-run experience

## GitHub Growth Mechanics

### Star CTA Strategy

Use a soft, useful CTA:

```text
If Luma Recorder helps you record lessons or training videos, a GitHub star helps more educators discover it.
```

Where to place it:

- bottom of README
- LinkedIn launch post
- Twitter/X thread final tweet
- Dev.to article conclusion
- release notes

### Issue Templates

Create three issue templates:

1. Bug report
2. Feature request
3. Teaching workflow request

Bug report fields:

- Windows version
- Luma Recorder version
- What you tried to record
- Expected result
- Actual result
- Steps to reproduce
- Screenshot/log if useful

Feature request fields:

- What teaching/training problem does this solve?
- Who would use it?
- What should the workflow feel like?
- Is there an example from another tool?

Teaching workflow request fields:

- What are you teaching?
- How long are your recordings?
- Do you need mic, webcam, system audio, or all three?
- What slows you down today?

### Discussion Prompts


```text

```

### Weekly Update Strategy

Every week for the first month, post:

- what shipped
- what users asked for
- what is next
- one useful teaching workflow
- thanks to contributors

Template:

```text
Luma Recorder weekly update

Shipped:
- ...

Feedback heard:
- ...

Next:
- ...

Question:
What would make recording lessons faster for you?
```

## Viral Hooks

### Show Your Setup

Prompt:

```text
Show your teaching setup.

Post a screenshot or short clip of how you record lessons:
- mic
- camera
- screen
- notes
- Luma Recorder settings
```

### Record Your Workflow Challenge

Challenge:

```text
Record a 60-second workflow lesson with Luma Recorder.

Rules:
- one concept
- one screen
- one clear explanation
- share the clip or screenshot
```

### Before/After Content

Examples:

- Before: long written explanation. After: 45-second screen lesson.
- Before: unclear bug report. After: narrated screen recording.
- Before: live training repeated every week. After: reusable onboarding clip.

### Encourage Users To Post Clips

CTA:

```text
If Luma Recorder helped you explain something faster, post a short clip and tag the repo.
```

## Positioning Angles

### 1. Educator Tool

Luma Recorder helps educators turn screen explanations into local lesson videos quickly, with voice, camera, and simple polish.

Hook:

```text
Record a lesson before the idea leaves your head.
```

### 2. Bug Reporting Tool

Luma Recorder helps teams turn unclear bug descriptions into narrated screen recordings that developers can understand faster.

Hook:

```text
Stop writing long bug reports. Show the issue in 60 seconds.
```

### 3. Content Creation Tool

Luma Recorder helps knowledge creators capture tutorials, workflows, and micro-lessons without setting up a heavy production stack.

Hook:

```text
Turn one workflow into a teachable clip.
```
